# Product Roadmap

## Phase 0 — Discovery and validation

Goal: Validate that the result feels specific and useful.

Deliverables:

- Finalize the first journey concept.
- Define 8–10 profile dimensions.
- Draft 40–50 candidate questions.
- Select 25–35 questions for prototype.
- Define 8–12 initial archetypes.
- Run manual assessments with at least 20 participants.
- Collect result accuracy feedback.
- Remove generic, flattering, or leading insights.

Exit criteria:

- At least 60% of participants say the main result is highly accurate.
- At least three insights consistently feel specific rather than generic.
- Question completion remains comfortable.

## Phase 1 — Technical prototype

Goal: Build the assessment engine without production infrastructure.

Deliverables:

- React and TypeScript PWA.
- Hard-coded or JSON-defined journey.
- Ranked answer interface.
- Local branching.
- Local scoring.
- IndexedDB persistence.
- Basic offline support.
- Simple result page.
- Developer simulator.

Exit criteria:

- Complete journey works in airplane mode.
- Refresh does not lose progress.
- Result is deterministic for the same answers.
- Automated unit tests cover scoring.

## Phase 2 — MVP

Goal: Release a reliable public beta.

Deliverables:

- Installable PWA.
- Anonymous-first flow.
- Downloaded journey package.
- Service worker and update management.
- Local result generation.
- Backend synchronization.
- Optional registration.
- Shareable result card.
- Basic analytics.
- Privacy and consent screens.
- Error monitoring.

Exit criteria:

- No critical answer-loss bugs.
- Offline completion tested on iPhone and Android.
- Synchronization reliability above 99% in test scenarios.
- Result feedback collection operational.

## Phase 3 — Closed beta

Goal: Improve the psychological quality and product experience.

Deliverables:

- 100–300 invited testers.
- Question performance dashboard.
- Drop-off analysis.
- Insight feedback analysis.
- Archetype distribution review.
- Revised scoring weights.
- Accessibility audit.
- Security review.
- Copy review.

Exit criteria:

- No archetype dominates due to scoring bias.
- Completion rate above 65%.
- Result resonance above target.
- Major accessibility issues resolved.

## Phase 4 — Version 1.0

Goal: Public launch.

Deliverables:

- Production-grade onboarding.
- Refined first journey.
- Cross-device account sync.
- Personal Map.
- Result history.
- Improved share cards.
- Terms, privacy, deletion flow.
- Operational dashboards.
- Backup and recovery procedures.

## Phase 5 — Additional journeys

Recommended order:

1. What really motivates me?
2. What happens to me under stress?
3. Why do I procrastinate?
4. How do I make decisions?
5. What are my blind spots?
6. What kind of life suits me?

Each journey should add evidence to shared dimensions.

## Phase 6 — Deeper product features

Potential features:

- Compare how I see myself with how friends see me.
- Invite a trusted person to answer about me.
- Longitudinal change tracking.
- Couple or friendship comparison.
- Personal reflection prompts.
- Private export.
- Multi-language support.
- Optional AI-guided exploration after the structured result.

These should not delay the MVP.
